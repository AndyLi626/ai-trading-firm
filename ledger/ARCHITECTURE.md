# Architecture — AI Trading Firm Platform
_Last updated: auto-generated by snapshot_capabilities.py_

## Core Principles
1. **Deterministic scripts first** — price/anomaly detection = pure Python, no LLM
2. **LLM = explanation layer only** — triggered after deterministic event, not polling full market
3. **delivery: none everywhere** — only manager-30min-report and infra-5min-report may use announce
4. **Budget enforced** — every LLM path goes through run_with_budget (70% warn / 85% degrade / 95% stop)
5. **config_guard governs** — no direct openclaw.json writes; proposal → review → apply
6. **chain_id tracing** — every event carries chain_id from trigger → scan → signal → proposal → verdict

## Event-Driven Intelligence Loop
```
market_pulse (15min) → anomaly_detector (5min)
  → emergency_trigger (dedup 10min, cap 20/h)
    → emergency_scan (1min poll, MediaBot)
      → strategy_hint (budget-gated, ResearchBot)
        → risk_review_lite (rules, no LLM)
          → event_proposals.json + risk_verdict.json
            → ManagerBot 30min report (EVENT DECISION LOOP)
```

## Routing Rules
- /pulse [symbols] → emergency_trigger.py → pending queue
- Cron anomaly → emergency_trigger.py (auto)
- All outputs → /tmp/oc_facts/ + memory/autonomy/YYYY-MM-DD/
- GCP signals → trading_firm.market_signals (chain_id, trigger tagged)
- No Telegram from bot scripts — only ManagerBot/InfraBot via delivery
